# Lakebase

Lakebase is Databricks managed PostgreSQL for operational workloads that need low-latency, transactional access.

It is designed for application backends that run next to Lakehouse data and governance controls.

## Why Lakebase

- PostgreSQL-compatible surface area for existing tools and ORMs
- managed operations with Databricks-native integrations
- support for modern database workflows, including autoscaling and branch-based development
- unified governance model with the rest of the Databricks platform

## What this section covers

- `lakebase/getting-started`: setup and first operational workflow.
- `lakebase/core-concepts`: architecture and capability model.
- `lakebase/development`: schema, connection, and lifecycle practices for app teams.

## Start here

For first implementation work, begin with `lakebase/getting-started`, then move to `lakebase/development` before production rollout.
